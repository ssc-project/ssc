SSC Parser

§Performance

The following optimization techniques are used:

  • AST is allocated in a memory arena (bumpalo) for fast AST drop
  • Short strings are inlined by CompactString
  • No other heap allocations are done except the above two
  • [oxc_span::Span] offsets uses u32 instead of usize

§Usage

The parser has a minimal API with two inputs and one return struct (ParserReturn).

let parser_return = Parser::new(&allocator, &source_text).parse();

§Visitor

See [ssc_ast::Visit] and [ssc_ast::VisitMut]
